## The Core Answer

**"Which completed audits can be trusted for production decision-making, and which are invalid because they rely on synthetic data?"**

### Trustworthy (HIGH confidence — code-proven, no data dependency)

**10+ tracks: TRACK-9A/B/C/D, TRACK-10D/E/F, TRACK-11, TRACK-14A, TRACK-15, TRACK-17, TRACK-18**

These audits analyzed code structure, formulas, weights, import graphs, consumer traces, and schema. Their conclusions are verifiable by reading source code and do not depend on the data in the database.

### Untrustworthy (LOW confidence — synthetic data foundation)

**TRACK-13, TRACK-14, EngineCalibrationReport.md, Top20Report.md, Bottom20Report.md, FactorAttributionReport.md, PercentileValidationReport.md, StockStoryExplainabilityReport.md**

These depend on data that was generated by `Math.random()` (candles) or `bounded(seed)` (engine inputs). Every number, ranking, histogram, and sector distribution in these reports is mathematically valid but factually meaningless.

### Mixed (MEDIUM confidence)

**TRACK-12** — The ROA weight change is code-proven and correct. The expected ranking impact estimates (~3-5 positions) are reasoned from generic distributions and are directionally plausible but numerically unvalidated.

## What Was Discovered

1. **`expand-market-coverage.ts`** generates 100% synthetic OHLCV candles using `Math.random()` for 505 stocks × 1250 days. If run, it populates the production DB with fake data.

2. **`generate-deliverables.ts`** generates 100% synthetic EngineInputs using `bounded(seed)` — a deterministic pseudo-random function. It produces the Top20Report, Bottom20Report, FactorAttributionReport, and PercentileValidationReport with fake data.

3. **`SectorDistributionEngine.ts`** uses hardcoded reference distributions — claimed to be "empirical Indian market data (2024-2025)" but implemented as static code constants. Not synthetic per se, but not live either.

4. **`generate500Stocks.ts`** provides real Indian ticker symbols with correct names/sectors — but ALL data for these stocks is synthetic.

5. **`run-research-validation.ts`** is the ONLY calibration script that uses real provider data (Yahoo) — but only for 7 symbols.

6. **Every API endpoint** (`/api/stockstory`, `/api/intelligence`, `/api/company`, `/api/discovery`) reads from DB tables that may contain synthetic data.

7. **The engine itself is sound.** All formulas are correct. All weights are reasonable. All code paths are traced. The problem is not the engine — it's the fuel.
