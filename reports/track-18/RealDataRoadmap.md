## Q7: Minimum Path to Real Data

---

### Step 1: Start PostgreSQL

| Action | Detail |
|--------|--------|
| **What** | Start PostgreSQL on localhost:5432 |
| **Why** | DB is currently unreachable (ECONNREFUSED) |
| **Effort** | ~1 minute (docker-compose up or pg_ctl start) |
| **Prerequisites** | PostgreSQL installed or Docker running |

---

### Step 2: Run `run-research-validation.ts` For All 505 Symbols

| Action | Detail |
|--------|--------|
| **What** | Extend `run-research-validation.ts` to process all 505 symbols from `generate500Stocks()` instead of just 7 research symbols |
| **Why** | This populates `daily_prices`, `feature_snapshots`, `factor_snapshots`, and `financial_snapshots` with REAL data from YahooProvider |
| **Effort** | ~1-2 days (rate limiting, error handling, batch processing) |
| **Prerequisites** | ProviderCoordinator with working Yahoo API key. Rate limit: ~1 request/second for free tier = ~8 minutes for 505 symbols. |
| **Key code** | `const RESEARCH_SYMBOLS = ["RELIANCE", "TCS", ...]` — expand to all 505 |

---

### Step 3: Run `calibrate_v2.ts`

| Action | Detail |
|--------|--------|
| **What** | Re-run calibration against the REAL DB |
| **Why** | Produces the first trustworthy EngineCalibrationReport with actual market distributions |
| **Effort** | ~5 minutes (script execution) |
| **Prerequisites** | Step 2 complete (real data in DB) |
| **What changes** | Stretch factor, risk dampening, sector weights will be calibrated from REAL Indian market data |

---

### Step 4: Re-run `generate-deliverables.ts` Against Real DB

| Action | Detail |
|--------|--------|
| **What** | Rewrite `generate-deliverables.ts` to READ from DB instead of using `bounded(seed)` synthetic inputs |
| **Why** | Produce the first trustworthy Top20Report, Bottom20Report, FactorAttributionReport |
| **Effort** | ~1 day (replace `buildInputs()` with DB queries) |
| **Prerequisites** | Step 2 complete |

---

### Step 5: Real Financial Snapshots

| Action | Detail |
|--------|--------|
| **What** | Replace `expand-market-coverage.ts` synthetic financial ratios with live data from ProviderCoordinator.getFinancials() |
| **Why** | PE, ROE, D/E, margins, etc. currently generated by `Math.random()` |
| **Effort** | ~3-5 days (provider integration, field mapping, validation) |
| **Prerequisites** | Working Upstox/Screener API keys. Rate limiting for 505 symbols × financial provider calls. |

---

### Step 6: Real Factor Snapshots

| Action | Detail |
|--------|--------|
| **What** | Run FactorEngine on real features (already computed from real daily_prices in Step 2) |
| **Why** | FactorEngine reads `feature_snapshots` — once those are real, factors become real |
| **Effort** | ~0 days (FactorEngine runs automatically as part of Step 2 pipeline) |

---

### Minimum Viable Path (Shortcut)

The absolute minimum to get real rankings:

1. Start PostgreSQL
2. Modify `run-research-validation.ts` to process all 505 symbols
3. Run it
4. Run `calibrate_v2.ts`

**Time to first real ranking: ~1 day of engineering + ~8 minutes of data fetching.**

This produces:
- ✅ Real daily prices (from Yahoo)
- ✅ Real technical features (FeatureEngine from real prices)
- ✅ Real factor scores (FactorEngine from real features)
- ✅ Real calibration report (calibrate_v2 from real distributions)
- ⚠️ Financial ratios still synthetic (from expand-market-coverage Math.random()) — unless Step 5 is also done

### Data Quality Checklist

| Check | Query |
|-------|-------|
| Prices are real | `SELECT symbol, close FROM daily_prices WHERE symbol='RELIANCE' ORDER BY trade_date DESC LIMIT 5` — compare with Yahoo Finance |
| Features computed correctly | `SELECT rsi, macd FROM feature_snapshots WHERE symbol='RELIANCE' ORDER BY trade_date DESC LIMIT 1` — sanity check values (RSI 0-100, MACD small number) |
| No synthetic ISINs | `SELECT COUNT(*) FROM symbols WHERE isin LIKE 'INE%' AND LENGTH(isin) = 12` — should be 0 after real data migration |
| All 505 symbols have data | `SELECT COUNT(DISTINCT symbol) FROM daily_prices` — should be 505 |
